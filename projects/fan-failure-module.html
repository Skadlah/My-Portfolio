Fan Failure Module

A PIC16F877A-based system for ensuring uninterrupted cooling in panels.

The Problem

In industrial control panels, cooling fans are critical for maintaining safe operating temperatures. If a primary fan fails, the internal temperature can rise rapidly, leading to component damage and costly downtime. A system is needed to detect this failure and automatically switch to a backup fan.

My Solution

I developed a 'Fan Failure Module' using a PIC16F877A microcontroller. The system monitors panel temperature and manages fan operations. Its primary function is to ensure uninterrupted cooling by automatically switching to a backup fan in case the primary fan fails, and then attempting an automatic recovery.

How It Works: The Technical Details

The logic is built around reliability and automatic recovery.

  • Temperature Monitoring: The PIC microcontroller continuously monitors the panel's temperature based on set thresholds.
  • Automatic Switching: A Relay Module is used for the physical switching. If the primary fan fails (which could be detected by a sensor or a rise in temperature despite the fan being 'on'), the PIC energizes the relay to switch the power to the backup fan.
  • Automatic Recovery: The system is designed to manage the fans to ensure the panel remains cool, potentially logging the failure and alerting a user, while automatically recovering the cooling function.

Tech Stack Used

  • Platform: PIC16F877A
  • Hardware: Relay Module, Temperature Sensors
  • Core Logic: Automatic failover switching
  • Software: C (MPLAB)
