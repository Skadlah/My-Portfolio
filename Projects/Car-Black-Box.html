Car Black Box System

A PIC-based system for real-time event logging and post-event analysis.

The Problem

In case of a vehicle incident, it's critical to have a reliable record of what was happening right before and during the event. A 'Black Box' needs to capture key data and store it securely in a way that survives the event and can be retrieved later for analysis.

My Solution

I developed a real-time Car Black Box using the **PIC16F877A** microcontroller. This system is designed to continuously record event IDs, timestamps, and user inputs (like button presses or sensor triggers). The data is displayed live on a **CLCD** (Character LCD)

How It Works: The Technical Details

The core of the system is data integrity and secure storage.

  • Persistent Storage: To ensure data is never lost (even on power failure), all logs are stored permanently in an **external EEPROM** module
  • Data Communication: The PIC microcontroller communicates with the EEPROM using the **I2C protocol, which is a robust, two-wire interface ideal for this application.
  • Data Retrieval: For post-event analysis, all stored data can be retrieved from the device by connecting it to a PC using **UART communication**. This allows for easy downloading and analysis of incident logs.

Tech Stack Used

  • Platform: PIC16F877A
  • Display: CLCD (Character LCD)
  • Storage: External EEPROM
  • Protocols: I2C, UART
  • Software: C (MPLAB)
