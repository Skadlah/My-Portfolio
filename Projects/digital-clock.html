Digital Clock System (PIC16F877A)

A dual-mode clock system using internal timers and seven-segment displays.

The Problem

Creating an accurate timekeeping device from scratch requires precise handling of timers and interrupts. This project was designed to master core microcontroller concepts: timer management, interrupt handling, and multiplexing displays (like Seven-Segment Displays - SSDs).

My Solution

I designed a dual-mode clock system using the PIC16F877A and four seven-segment displays (SSDs). The system can operate in two distinct modes: a static display mode for testing, and a dynamic mode for real-time clock functionality.

How It Works: The Technical Details

The project's complexity lies in its dual functionality and precise timing.

  • Static Display Mode: In this mode, the system accepts user inputs to display preset time values on the SSDs. This is useful for testing the display connections and multiplexing logic.
  • Dynamic (Real-Time) Mode: This is the functional clock mode. It uses the PIC's internal timers and interrupts to generate accurate one-second ticks. The logic updates the time variables (seconds, minutes, hours) and refreshes the four SSDs.
  • User Input: Mode selection (static vs. dynamic) and setting the time are handled via user inputs (buttons).

Tech Stack Used

  • Platform: PIC16F877A
  • Display: 4 x Seven-Segment Displays (SSDs)
  • Core Concepts: Internal Timers, Interrupts, Display Multiplexing
  • Software: C (MPLAB)
